| success/failure state -> partial diagnostics exposure | partial data must not blur the binary run status |
| core report -> CLI renderer | machine-readable shape must remain canonical even if text rendering differs |

| T-17-01-02 | I | failure reporting semantics | mitigate | expose partial diagnostics only as supplementary failure fields and forbid `partial_success` |
| T-17-01-03 | D | execution/reporting layering | mitigate | build reporting types directly from Phase 16 execution results rather than inventing a second execution interpretation layer |
</threat_model>
