<objective>
Polish the current `capture` command into a stable product surface by locking clear success and failure output for both human shell use and machine-readable automation.
</objective>

<scope>
This plan covers result rendering, success output shape, failure output shape, and stable CLI-facing diagnostics for the existing one-shot capture flow. It does not redesign artifact path control beyond the minimum result-shape support needed here.
</scope>
