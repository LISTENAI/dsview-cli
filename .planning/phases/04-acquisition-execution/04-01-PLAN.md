<objective>
Resolve the clean finite-capture success signal up front, confirm this machine is ready for hardware-backed Phase 4 work, and then add the first real acquisition execution path for `DSLogic Plus` through the existing sys/core/cli seam.
</objective>

<scope>
This plan covers the success-signal decision, local execution preflight, the happy-path acquisition pipeline, and the minimal CLI entry point needed to trigger it. It does not add VCD export, metadata artifacts, or broader productization UX from later phases.
</scope>
