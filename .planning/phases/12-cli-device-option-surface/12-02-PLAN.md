## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI input -> token resolver | Friendly token strings and threshold numbers are untrusted until they map back to known stable IDs or numeric ranges |
| CLI resolver -> `dsview-core` validator | The CLI must construct a full request from current values plus overrides without silently changing the user's explicit choices |
| Phase 12 validation -> capture runtime | Phase 12 may validate option selections, but it must not imply or perform the Phase 13 hardware-apply step |
