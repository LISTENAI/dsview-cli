<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust → C bridge | Safe Rust calls into `bridge_runtime.c` and must not receive borrowed native pointers or invalid lifetimes back |
| C bridge → DSView config APIs | `GVariant` payloads and pointer-valued lists come from DSView and are untrusted until copied/validated |
| Discovery helper → active device state | Temporary operation-mode switching can mutate live device state if restore logic is incomplete |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-10-01-03 | S/T | Channel-mode metadata mapping | mitigate | Copy explicit driver metadata fields instead of deriving machine meaning from human labels |
</threat_model>
