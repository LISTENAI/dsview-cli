<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust -> decode runtime loader | Caller-supplied runtime/decoder paths must not collapse into vague loader failures |
| decode runtime -> upstream decoder metadata | Upstream decoder structs contain nested pointers and dynamic lists that cannot be exposed directly to safe Rust |
| project runtime layering | Decode support must not silently expand the dependency contract of the shipped capture runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01-01 | D | decode runtime bring-up | mitigate | Separate runtime artifact, explicit path-first init, and distinct loader/init/decode-dir errors |
| T-14-01-02 | I | metadata snapshot boundary | mitigate | Deep-copy strings and nested arrays before Rust sees them, plus explicit free helpers |
| T-14-01-03 | T | runtime layering | mitigate | Keep decode sources out of the existing capture runtime target and lock the separate artifact name with tests |
</threat_model>
